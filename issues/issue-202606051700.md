あなたはエキスパートのフルスタックエンジニアです。
以下の要件と技術スタックに従って、MisskeyのPWAクライアントアプリケーションを新規作成してください。複雑なプロジェクトになるため、一度にすべてを実装せず、指定したステップごとに実装と確認を進めてください。

# プロジェクト概要
MisskeyのAPIと通信し、タイムラインの閲覧やノートの投稿ができるPWAクライアントを開発します。

# 技術スタック
- フロントエンド: React (Viteを使用)
- バックエンド (APIルーティング/SSR): Hono
- デプロイ先: Cloudflare (Cloudflare Pages + Functions 構成を想定)
- ユニット/コンポーネントテスト: Vitest
- E2Eテスト: Playwright
- PWA対応: vite-plugin-pwa 等を使用

# 必須ライブラリと参考ドキュメント
実装の際は、必ず以下の公式ライブラリとドキュメントをベースにしてください。
- 公式SDK: https://github.com/misskey-dev/misskey/tree/develop/packages/misskey-js
- Misskey APIドキュメント: https://misskey.io/api-doc

# 必須要件
1. Misskey連携 (`misskey-js` の活用)
   - MiAuthを用いた認証フローの実装
   - `misskey-js` の `Stream` クラスを用いたタイムラインのリアルタイム更新（WebSocket）
   - `misskey-js` の APIクライアントを用いたノートの投稿・取得
2. PWA対応
   - manifest.jsonの適切な設定、オフラインキャッシュ、インストールプロンプト対応
3. Honoの活用
   - Reactの配信、またはAPIプロキシ/認証コールバック処理としてのルーティングをHonoで構築
   - `hono/cloudflare-pages` などのアダプタを使用し、Cloudflare環境で動くよう最適化
4. テスト要件
   - Vitest: `misskey-js` を利用した主要なロジックやReactコンポーネントの単体テスト
   - Playwright: アプリの起動、ログインのモック、タイムラインの表示確認などの基本的なE2Eフロー
5. デプロイ設定
   - Cloudflare Pagesへデプロイするための `wrangler.toml` およびビルドスクリプトの整備

# 実装ステップ
以下の順番で実装を進めてください。各ステップが終わるごとに進捗と生成したコードを報告し、次のステップに進んでください。

- [Step 1] プロジェクトの初期セットアップ: Vite + React + Honoの構成を構築し、ローカル開発環境（`@hono/vite-dev-server` 等）を整える。
- [Step 2] PWAとテストのセットアップ: `vite-plugin-pwa` の設定、VitestおよびPlaywrightの初期設定を行う。
- [Step 3] Misskey SDK導入と認証: `misskey-js` をインストールし、MiAuthのフローを構築する。
- [Step 4] フロントエンド実装: Reactと `misskey-js` を使い、タイムライン画面（ストリーミング含む）と投稿フォームのUIを作成する。
- [Step 5] テストの記述: Step 4で作ったUIとロジックに対して、VitestとPlaywrightのテストケースを記述し、パスさせる。
- [Step 6] Cloudflareデプロイ準備: ビルドコマンドやデプロイ用の構成ファイル（wrangler.toml等）を完成させる。
